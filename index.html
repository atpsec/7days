Başlangıç komutları

Pratik araç örnekleri

Bu komutları yalnızca kendi repolarında ve izinli lokal lab ortamlarında kullan.

Bandit JSON çıktısı
pip install bandit
bandit -r . -f json -o bandit-report.json
Bandit SARIF çıktısı
pip install bandit
bandit -r . -f sarif -o bandit-results.sarif
GitHub Actions: Bandit + SARIF upload
name: security-scan

on:
  push:
    branches: [ main ]
  pull_request:

permissions:
  contents: read
  security-events: write

jobs:
  bandit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install bandit
      - run: bandit -r . -f json -o bandit-report.json || true
      - run: bandit -r . -f sarif -o bandit-results.sarif || true
      - uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: bandit-results.sarif

Şablonlar

Günlük ve haftalık takip

Günlük çalışma şablonu

  • 10 dk: Dün öğrendiğin 5 kavramı notsuz yaz.
  • 20 dk: Tek konuyu sade tanım + örnekle öğren.
  • 10 dk: 5 aktif hatırlama sorusuna cevap ver.
  • 10 dk: Feynman açıklaması yaz.
  • 30–60 dk: Lab uygula.
  • 10–20 dk: Raporu Markdown/HTML/JSON/SARIF olarak kaydet.
  • 5 dk: “Bugün ne öğrendim, neyi karıştırıyorum?” notu yaz.

Haftalık değerlendirme formu

  • Konuları notsuz açıklayabiliyor muyum?
  • GitHub Actions workflow yazabiliyor muyum?
  • Bulguları risk seviyesine göre ayırabiliyor muyum?
  • JSON/SARIF/HTML rapor üretebiliyor muyum?
  • Pipeline fail/pass kararını gerekçelendirebiliyor muyum?
  • Portföye eklenebilir çıktı ürettim mi?

Risk önceliklendirme modeli

  • Severity (önem): Low, Medium, High, Critical
  • Exploitability (sömürülebilirlik): düşük/orta/yüksek
  • Asset criticality (varlık kritiklik değeri): düşük/orta/yüksek
  • Remediation effort (düzeltme eforu): kolay/orta/zor
  • Blocking policy: High/Critical bulguda pipeline durur.
